
# 🎮 لعبة اسم، حيوان، نبات، جماد، بلاد - WebRTC Edition

نسخة رقمية وتفاعلية بالكامل من اللعبة الجماعية الكلاسيكية الشهيرة. تم تصميم هذا المشروع ليكون تطبيق ويب من صفحة واحدة (SPA) يعمل بالكامل على المتصفح (Client-Side) ويدعم اللعب الجماعي في الوقت الفعلي (Real-time Multiplayer) دون الحاجة لأي خادم خلفي (Serverless) أو قاعدة بيانات، وذلك بالاعتماد على تقنية **WebRTC**.

## 🚀 المميزات الأساسية (Features)

* **لعب جماعي مباشر (Peer-to-Peer):** اتصال مباشر بين اللاعبين عبر المتصفحات بدون تأخير، باستخدام مكتبة `PeerJS`.
* **بدون خوادم (Serverless):** لا يتطلب المشروع أي إعدادات لقواعد البيانات (مثل Firebase) أو خوادم خلفية (Backend).
* **نظام الإيقاف السريع (The Buzzer):** بمجرد أن ينهي أول لاعب إجاباته ويضغط على زر "انتهيت!"، يتم تعطيل حقول الإدخال فوراً عند جميع اللاعبين في نفس اللحظة لمنع الغش.
* **اختيار تلقائي للحرف:** يقوم النظام باختيار حرف عربي عشوائي في بداية كل جولة لبدء التحدي.
* **تصميم عصري (Modern UI):** واجهات نظيفة تعتمد على تأثيرات `Glassmorphism`، وتدعم جميع الشاشات (الهواتف، الأجهزة اللوحية، الحواسيب).
* **دعم كامل للغة العربية (RTL):** الواجهة والتجربة مصممة بالكامل من اليمين لليسار.

## 🛠️ التقنيات المستخدمة (Tech Stack)

* **الواجهة الأمامية (Frontend):** HTML5, CSS3, Vanilla JavaScript.
* **الشبكات واللعب الجماعي:** [PeerJS](https://peerjs.com/) (WebRTC) لإدارة الاتصال المباشر ونقل البيانات (Data Channels).
* **الاستضافة (Hosting):** GitHub Pages (بما أن المشروع عبارة عن ملفات ثابتة Static Files).

### واجهات التطبيق (SPA Sections)
يتم التنقل بين هذه الأقسام برمجياً (عن طريق إخفاء/إظهار عناصر DOM):
 1. section-home: شاشة تسجيل الدخول وإنشاء/الانضمام للغرف.
 2. section-lobby: غرفة الانتظار لمعرفة من انضم وتأكيد الاستعداد.
 3. section-game: ساحة اللعب الرئيسية وحقول الإدخال.
 4. section-score: لوحة عرض النتائج والإجابات بعد كل جولة.
## ⚙️ آلية عمل اللعب الجماعي (How WebRTC Works Here)
بما أننا لا نستخدم خادماً، يتم تقسيم أدوار اللاعبين إلى:
 * **المضيف (The Host):** اللاعب الذي يضغط على "إنشاء غرفة". يتم منحه Peer ID يعمل كرمز للغرفة. المضيف هو من يستقبل الاتصالات، يدير الجولة، يختار الحرف العشوائي، ويستقبل إجابات الجميع لعرضها.
 * **الضيف (The Guest):** اللاعب الذي ينضم باستخدام رمز الغرفة. يقوم بإرسال إجاباته وإشارات الاستعداد للمضيف فقط.
 * **توزيع البيانات (Broadcasting):** عندما يحدث أي تغيير (مثل انضمام لاعب جديد أو ضغط أحدهم على زر الإيقاف)، يقوم المضيف بتعميم هذه الإشارة لجميع الضيوف المتصلين به.
## 💻 طريقة التشغيل (How to Play / Run)
بما أن المشروع جاهز للعمل على **GitHub Pages**:
 1. اذهب إلى إعدادات المستودع (Settings) في GitHub.
 2. انتقل إلى قسم **Pages** من القائمة الجانبية.
 3. في قسم Build and deployment، اختر الـ Branch الرئيسي (غالباً main أو master) واضغط حفظ.
 4. خلال دقائق، سيوفر لك GitHub رابطاً مباشراً للعبة.
 5. شارك الرابط مع أصدقائك! يدخل أحدهم كـ "مضيف" ويعطي البقية "رمز الغرفة" لينضموا إليه.
